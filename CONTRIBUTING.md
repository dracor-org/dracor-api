# Contributing to the DraCor API

Thank you for your interest in contributing to the DraCor API. DraCor is a
community-driven project, and we welcome contributions from developers,
researchers, and users alike—no matter how large or small.

## Ways to Contribute

There are several ways to get involved:

- **Open an issue**
  Please use [GitHub issues](https://github.com/dracor-org/dracor-api/issues) to
  report bugs, ask questions, or suggest enhancements. Clear descriptions and
  examples are greatly appreciated.

- **Submit a pull request**
  We welcome pull requests for bug fixes, new features, performance
  improvements, and documentation updates. If you are planning a larger change,
  we recommend opening an issue first to discuss it.

- **Join the discussion**
  [GitHub Discussions](https://github.com/dracor-org/dracor-api/discussions) is
  the right place for broader questions, ideas, or conversations about the
  future direction of the API.

## Getting Started

For instructions on setting up the project locally, running the API, or
executing tests, please refer to the [README](README.md).

If anything is unclear or missing, feel free to ask—we are happy to help.

See https://dracor.org/doc/get-in-touch for ways to get in touch with the team.

## Pull Request Guidelines

- Keep changes focused and well scoped.
- Follow existing coding and documentation conventions where possible.
- Include tests or documentation updates when relevant.
- Be prepared to revise your contribution based on feedback.

## Code of Conduct

All contributors are expected to interact respectfully and constructively. By
participating in this project, you agree to uphold a welcoming and inclusive
environment.
